# CountBot

> 面向中文用户的轻量级 AI Agent 运行中枢，连接大模型、消息渠道、工具调用与工作流，让 AI 从“对话回复”进一步进入“任务执行”。

## 项目简介

CountBot 是一个面向本地部署和长期运行场景的 AI Agent 项目。项目围绕“大模型接入、Prompt 编排、工具调用、消息渠道、任务调度、运行记忆”等能力进行设计，支持将 AI 能力接入到不同 IM 渠道和业务流程中。

该项目适合作为个人 AI 助手、团队自动化机器人、客服/运营助手、内容生成助手、工作流执行中枢等场景的基础框架。

## 项目背景

普通的大模型应用通常只停留在单轮或多轮对话层面，缺少稳定的任务调度、渠道接入、工具调用和长期运行能力。CountBot 通过统一的 Agent 运行框架，将模型能力、消息入口、工具执行和业务流程进行整合，解决 AI 应用在实际落地中的以下问题：

- 多模型接入成本高，不同模型接口难以统一管理
- AI 只能对话，缺少调用工具和执行业务动作的能力
- 消息入口分散，难以同时支持 Web、微信、企业微信等多个渠道
- 长期任务缺少队列、重试、去重和失败处理机制
- Prompt、角色、工作流和工具配置分散，不利于维护和扩展

## 核心功能

### 1. 多模型接入

项目支持对接不同大模型服务，可根据任务类型选择不同模型策略，便于后续扩展更多模型提供商。

实现价值：

- 降低模型切换成本
- 支持不同场景使用不同模型
- 提升系统的可扩展性和稳定性

### 2. Agent 工具调用

系统支持将外部能力封装为工具，由 Agent 在对话或任务执行过程中按需调用。

典型工具能力包括：

- Web 查询
- 文件处理
- Shell 命令执行
- 外部 API 调用
- 业务系统接口调用
- 自定义 Skill 扩展

实现价值：

- 让 AI 不只生成文本，还可以执行具体动作
- 支持将复杂任务拆解为多个工具调用步骤
- 方便后续接入更多业务能力

### 3. Prompt 模板与角色配置

项目支持通过 Prompt 模板和角色配置管理 Agent 行为，可针对不同业务场景配置不同的系统提示词、角色设定和执行策略。

实现价值：

- 降低 Prompt 维护成本
- 便于复用不同 Agent 角色
- 提高回复稳定性和任务执行一致性

### 4. 企业级消息队列机制

项目围绕长期运行任务设计了消息队列能力，支持任务优先级、消息去重、失败重试和死信队列等机制。

核心能力包括：

- 优先级调度
- 消息去重
- 失败重试
- 死信队列
- 异常任务隔离

实现价值：

- 提升任务处理稳定性
- 避免重复消息导致多次执行
- 防止异常任务阻塞主流程
- 适合长期运行的 AI Agent 服务

### 5. 多渠道消息接入

项目支持将 AI Agent 接入不同消息渠道，使用户可以通过不同入口与 Agent 交互。

可扩展渠道包括：

- Web UI
- 微信
- 企业微信
- 飞书
- 钉钉
- Telegram
- QQ

实现价值：

- 支持多端统一交互
- 降低用户使用门槛
- 便于接入真实业务场景

### 6. 工作流编排

项目支持将复杂任务拆解为多个步骤，通过工作流方式组织模型调用、工具调用和结果输出。

适用场景：

- 内容生成流程
- 数据查询流程
- 自动化办公流程
- 客服应答流程
- 运营消息处理流程

### 7. 运行记忆与上下文管理

系统支持对会话上下文和历史信息进行管理，使 Agent 在长期运行过程中能够保持一定的连续性。

实现价值：

- 提升多轮对话体验
- 降低重复输入成本
- 让 Agent 更适合长期任务处理

## 技术栈

> 具体技术栈可根据你的实际代码进行调整。

- 后端：Python / FastAPI / 异步任务处理
- 前端：Vue / Vite / TypeScript
- 数据存储：SQLite / MySQL / Redis
- AI 能力：LLM API / 多模型适配 / Prompt 模板
- 工程能力：消息队列、任务调度、日志管理、配置管理
- 部署方式：Docker / Docker Compose / Linux 服务器部署

## 系统架构

```text
用户 / 外部系统
      │
      ▼
多渠道入口层
Web / 微信 / 企业微信 / 飞书 / Telegram / QQ
      │
      ▼
消息处理层
消息解析 / 去重 / 队列调度 / 重试 / 死信处理
      │
      ▼
Agent 编排层
角色配置 / Prompt 模板 / 工作流 / 上下文管理
      │
      ▼
模型与工具层
LLM 调用 / 工具调用 / Skill 扩展 / 外部 API
      │
      ▼
结果输出层
文本回复 / 文件输出 / 多渠道推送 / 任务结果记录
```

## 项目亮点

- 设计多模型适配层，统一不同模型调用方式，降低模型切换和扩展成本
- 封装 Agent 工具调用能力，使系统支持外部 API、文件、Shell、自定义 Skill 等任务执行能力
- 设计 Prompt 模板和角色配置机制，提高不同业务场景下 Agent 行为的可控性
- 引入消息队列思想，支持任务优先级、去重、重试和死信队列，提升系统长期运行稳定性
- 支持多渠道消息接入，可扩展微信、企业微信、飞书、钉钉、Telegram 等输出入口
- 支持工作流编排，将复杂任务拆分为多个模型调用和工具调用步骤，提高任务处理能力
- 项目结构清晰，便于后续扩展为个人助手、企业知识助手、客服机器人或自动化办公平台

## 快速开始

### 1. 克隆项目

```bash
git clone https://github.com/yeyang520/Countbot.git
cd Countbot
```

### 2. 安装依赖

后端依赖：

```bash
cd backend
pip install -r requirements.txt
```

前端依赖：

```bash
cd frontend
npm install
```

### 3. 配置环境变量

在项目根目录或后端目录下创建 `.env` 文件：

```env
LLM_API_KEY=your_api_key
LLM_BASE_URL=your_base_url
REDIS_HOST=localhost
REDIS_PORT=6379
```

### 4. 启动项目

启动后端：

```bash
cd backend
python main.py
```

启动前端：

```bash
cd frontend
npm run dev
```

如果项目提供 Docker Compose，可以使用：

```bash
docker compose up -d
```
