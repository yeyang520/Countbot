# v0.9.0 发布说明

## 版本概览

`v0.9.0` 的主题是"外部能力扩展与知识库集成"。

这个版本新增了 MCP 客户端模块与 Wiki 知识库模块，为 CountBot 扩展了外部工具集成与本地知识检索两大核心能力。同时新增 WebSocket 状态广播、API Key 轮换、心跳问候优化等多项改进，使系统在可扩展性、稳定性和成本控制方面均有显著提升。

## 核心更新

### 1. MCP 客户端模块

`v0.9.0` 新增 MCP（Model Context Protocol）客户端模块，为 CountBot 扩展外部工具与服务集成能力。默认关闭，面向有个性化需求的高级用户：

* 支持多 MCP 服务器连接与健康检查
* 自动工具发现与参数校验
* 连接状态实时同步至前端
* 完善的错误处理与超时重试机制
* 新增 MCP 管理面板，支持服务器配置、连接状态查看与工具列表浏览

### 2. Wiki 知识库模块

`v0.9.0` 内置基于 BM25 的全文搜索引擎，为 Agent 提供本地知识库检索能力：

**核心能力：**

| 能力 | 说明 |
| --- | --- |
| BM25 全文搜索 | 支持标题、标签与内容的全文检索，按相关性排序 |
| 批量获取 | 一次请求获取多个文档，减少重复调用 |
| 相关性过滤 | 自动过滤得分低于阈值的结果，提升回答质量 |
| LRU 缓存 | 缓存容量 128 个文档，重复查询性能提升 10-100 倍 |

**适用场景：**

* 企业文档、产品手册、技术规范等本地知识库管理
* Agent 需要参考固定资料回答用户问题的场景
* 需要快速检索大量 Markdown 文档的场景

### 3. WebSocket 状态广播

`v0.9.0` 新增 WebSocket 状态广播模块，实现服务端与前端实时联动：

* MCP 连接/断开状态实时推送至前端会话
* 多会话并发广播，支持多终端同步
* 轻量连接池与心跳保活机制
* 与现有 WebSocket 聊天系统无缝集成

### 4. 心跳问候逻辑简化

`v0.9.0` 简化了主动问候模块的决策流程：

* 去掉冗余的 LLM 决策层，问候触发从 2 次调用降为 1 次
* 前置检查在准备阶段完成（免打扰时段、随机时间点、空闲检测、每日上限）
* 减少 LLM 调用成本，保持核心策略不变

### 5. API Key 轮换机制

`v0.9.0` 新增 API Key 轮换与故障转移能力：

**核心特性：**

| 特性 | 说明 |
| --- | --- |
| 线程安全轮换器 | 支持多 key 自动切换，线程安全 |
| 自动故障转移 | 遇到 401/429 错误时自动重试并切换 key |
| 多 key 配置 | 支持在 Provider 配置中设置多个 API Key |

**改进效果：**

* 提升高并发场景下的可用性
* 避免因单 key 限流导致的服务中断
* 无需手动切换 key，系统自动管理

### 6. 前端面板与组件优化

`v0.9.0` 对前端管理面板与组件做了多项优化：

* 新增 MCP 管理面板，支持服务器增删改查与连接管理
* 新增 Wiki 知识库面板，支持文档编辑、搜索与预览
* 组件模块化拆分，降低耦合度
* 国际化支持进一步完善
* 优化模型预设界面，配置操作更顺手

### 7. 工具注册与初始化优化

`v0.9.0` 对工具注册与初始化流程做了系统性整理：

* 内置工具统一注册入口，启动时自动加载
* 工具初始化流程更清晰，提升配置一致性
* MCP 工具自动注册至 Agent 可用工具列表

### 8. 兼容性改进

`v0.9.0` 优化了多个边缘场景的兼容性：

* 改进 send_media 工具网页版兼容性，非频道会话只返回文本，由前端处理文件渲染
* 优化文件校验逻辑，提升上传稳定性
* 优化 StreamChunk 错误处理，认证错误直接抛出供上层处理
* 支持在 Web UI 预览图片和下载附件

## 适用场景

`v0.9.0` 特别适合以下场景：

* 需要接入外部 MCP 服务扩展 AI 能力边界
* 需要本地知识库检索，让 Agent 回答更准确
* 同时使用多个 API Key，需要自动轮换与故障转移
* 对系统稳定性和成本控制有较高要求
* 希望获得更实时的服务状态反馈
* 需要 MCP、Wiki 可视化管理面板

## 升级提示

* MCP 服务器需要在配置中手动添加，首次使用请参考 [MCP 配置文档](/docs/advanced/mcp)
* Wiki 知识库文档存放在 `workspace/wiki/` 目录，支持 Markdown 格式
* API Key 轮换功能需要在 Provider 配置中设置 `api_keys` 字段
* 心跳问候逻辑简化后，问候频率与之前保持一致，无需调整配置
* 建议在生产环境升级前先在测试环境验证 MCP 与 Wiki 功能
